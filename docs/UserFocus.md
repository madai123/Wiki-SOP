# User Focus Summary

用途：给 Daily Paper Watch 和其他轻量自动任务快速判断当前关注什么，避免每次读取完整 wiki。

更新原则：保持简短，只记录近期偏好、关注主题和筛选口径。不要写成长篇 synthesis。

公开仓库建议把本文件保留为模板；真实研究偏好可放在私有副本中。

> [!note] 当前关注
> - [focus area 1]
> - [focus area 2]
> - [method / task / dataset / benchmark of interest]
> - [topic to watch]

> [!tip] 优先推荐
> - 与当前 wiki 中已有 Concept / Entity / Gap / Synthesis 能形成延续关系的论文
> - 新 benchmark、新 dataset、新 system 或新的评测维度
> - 能补充已有 synthesis 的新证据
> - 能支持或反驳已有 gap / hypothesis 的论文
> - 具备明确 arXiv / OpenReview / DOI / publisher URL 等可核验来源的论文

> [!warning] 降低优先级
> - 与当前关注方向关系弱的论文
> - 只有 buzzword、没有清晰任务或实验证据的论文
> - 已经明显入库或已在近期 Daily Paper Watch 中推荐过的论文
> - 缺少可核验来源编号的条目

## Recent Threads

- Daily Paper Watch 需要覆盖公开论文来源和基于当前关注点临时生成的 query。
- 每日报告默认生成本地 Obsidian / Markdown 日报；如需外部发布，正文应与本地版本保持一致。
- 候选论文未入库前，不创建 `[[paper_id]]` 内部反链。
- 自动任务优先读这个短摘要，而不是每次全量读 wiki。

## Update Notes

- 这个文件应由近期问题、明确偏好和已确认方向更新。
- 每次更新尽量控制在 1 屏以内。
- 如果某个方向已经稳定成体系，应沉淀到 `wiki/synthesis/`，这里仅保留短引用或一句话提示。
